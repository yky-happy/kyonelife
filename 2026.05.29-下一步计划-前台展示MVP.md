# kyonelife 下一步计划：前台展示 MVP

> 日期：2026-05-29
> 结论：后台核心写作链路已经基本具备，下一步应优先做前台展示 MVP，让项目形成“后台发布 -> 前台可读”的博客闭环。

## 一、当前项目状态

| 模块 | 当前状态 | 判断 |
| --- | --- | --- |
| 后端基础设施 | 已完成 | Spring Boot、MyBatis Plus、Sa-Token、Knife4j、统一响应、异常处理、CORS 等已落地 |
| 后台登录 | 已完成基础链路 | 有登录接口、Pinia Token、axios 拦截器、登录页和路由守卫 |
| 标签管理 | 已完成端到端 CRUD | 后端接口、前端页面、分页搜索、新增编辑删除均已具备 |
| 合集管理 | 已完成端到端 CRUD | 可作为文章组织结构使用 |
| 文章管理 | 已基本完成 MVP | 后端 `ArticleController` / `ArticleServiceImpl` 和后台列表、编辑页、Markdown 编辑器已存在 |
| 评论、轮播、系统管理 | 以页面骨架为主 | 暂时不应作为主线 |
| 前台 `blog-web` | 仍是 Vite 初始页面 | 还没有路由、请求封装、文章列表、文章详情 |
| 前台公开 API | 目录已建但无实现 | `com.yky.blog.api` 包为空，需要新增前台接口 |
| 数据库初始化脚本 | 有明显问题 | `init.sql` 创建 `kyonelife`，但 `USE kyonelife_blog`，与 `application.yml` 不一致 |

当前不建议继续横向补后台管理模块，也不建议马上做 AI、评论、动态权限。最有价值的下一步，是让后台已经能发布的文章在前台真正展示出来。

## 二、阶段目标

下一阶段建议命名为：

**阶段 5：前台展示 MVP**

目标是跑通个人博客最小闭环：

```text
后台登录 -> 写文章 -> 发布文章 -> 前台首页看到文章 -> 点击进入文章详情
```

阶段完成后的验收结果：

1. 后台可以发布文章。
2. 前台首页只展示已发布文章。
3. 点击文章可以进入详情页阅读正文。
4. 前台可以按标签查看文章。
5. 前台可以按合集查看文章。
6. 前台可以按年月查看归档。
7. 手机端和桌面端都能正常浏览。

## 三、为什么现在做前台展示

当前更合理的依赖顺序是：

```text
后台登录
  -> 标签 / 合集 CRUD
  -> 文章 CRUD
  -> 前台文章列表 / 详情
  -> 评论
  -> 轮播 / 网站配置
  -> 图片上传 / 搜索 / AI
  -> 动态权限 / 部署优化
```

项目已经走到“文章 CRUD”。如果继续做评论、轮播、管理员、角色、网站配置，会让后台功能越来越多，但博客仍然没有面向读者的核心价值。

## 五、前置修复

### 5.1 修复数据库脚本库名不一致

当前 `blog/sql/init.sql` 顶部是：

```sql
CREATE DATABASE IF NOT EXISTS kyonelife DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE kyonelife_blog;
```

但 `blog/src/main/resources/application.yml` 连接的是：

```yaml
jdbc:mysql://localhost:3306/kyonelife
```

建议统一为：

```sql
CREATE DATABASE IF NOT EXISTS kyonelife DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE kyonelife;
```

这个问题必须先修，否则新环境初始化数据库时会直接失败。

### 5.2 回归后台文章管理

前台开发前，先做一次后台文章链路回归：

- 登录后台
- 新建一篇草稿
- 编辑标题、摘要、标签、合集
- 发布文章
- 下架文章
- 再次发布文章
- 删除一篇测试文章

如果这条链路还有问题，先修好再进入前台展示。

## 六、后端开发计划

### 6.1 包结构

已有空目录：

```text
blog/src/main/java/com/yky/blog/api
blog/src/main/java/com/yky/blog/api/controller
blog/src/main/java/com/yky/blog/api/service
blog/src/main/java/com/yky/blog/api/service/impl
blog/src/main/java/com/yky/blog/api/dto
```

建议继续使用 `api` 包作为前台公开接口包，不要把前台接口放到 `admin` 包里。

### 6.2 新增 VO

建议新增：

```text
blog/src/main/java/com/yky/blog/api/vo/ArticleCardVO.java
blog/src/main/java/com/yky/blog/api/vo/ArticleWebDetailVO.java
blog/src/main/java/com/yky/blog/api/vo/ArchiveVO.java
blog/src/main/java/com/yky/blog/api/vo/CollectionCardVO.java
```

MVP 可以复用后台 `TagVO`，但文章列表和详情建议使用前台专用 VO，避免把后台字段暴露给前台。

文章列表 `ArticleCardVO` 建议字段：

| 字段 | 说明 |
| --- | --- |
| id | 文章 ID |
| title | 标题 |
| cover | 封面 |
| summary | 摘要 |
| collectionId | 合集 ID |
| collectionName | 合集名称 |
| tags | 标签列表 |
| isStick | 是否置顶 |
| viewCount | 阅读量 |
| createTime | 发布时间 |

文章详情 `ArticleWebDetailVO` 建议字段：

| 字段 | 说明 |
| --- | --- |
| id | 文章 ID |
| title | 标题 |
| cover | 封面 |
| summary | 摘要 |
| content | HTML 内容 |
| contentMd | Markdown 原文，MVP 可返回给前端渲染 |
| collectionId | 合集 ID |
| collectionName | 合集名称 |
| tags | 标签列表 |
| keywords | SEO 关键词 |
| isOriginal | 是否原创 |
| originalUrl | 转载原文链接 |
| viewCount | 阅读量 |
| createTime | 发布时间 |

### 6.3 新增前台 Article API

建议新增：

```text
blog/src/main/java/com/yky/blog/api/controller/ArticleApiController.java
blog/src/main/java/com/yky/blog/api/service/ArticleApiService.java
blog/src/main/java/com/yky/blog/api/service/impl/ArticleApiServiceImpl.java
```

接口建议：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/article/page` | 前台文章分页 |
| GET | `/api/article/{id}` | 前台文章详情 |
| GET | `/api/article/archive` | 文章归档 |

分页参数建议：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| page | int | 当前页，默认 1 |
| size | int | 每页数量，默认 10 |
| keyword | String | 关键词，MVP 可选 |
| tagId | Long | 标签筛选 |
| collectionId | Long | 合集筛选 |

查询规则：

```text
status = 1
order by is_stick desc, create_time desc
```

重点：前台接口必须只返回已发布文章，不能泄漏草稿和已下架文章。

### 6.4 新增前台 Tag API

建议新增：

```text
blog/src/main/java/com/yky/blog/api/controller/TagApiController.java
blog/src/main/java/com/yky/blog/api/service/TagApiService.java
blog/src/main/java/com/yky/blog/api/service/impl/TagApiServiceImpl.java
```

接口建议：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/tag/list` | 标签列表 |

MVP 可以返回全部标签。后续再考虑只返回有已发布文章的标签，并补充文章数量。

### 6.5 新增前台 Collection API

建议新增：

```text
blog/src/main/java/com/yky/blog/api/controller/CollectionApiController.java
blog/src/main/java/com/yky/blog/api/service/CollectionApiService.java
blog/src/main/java/com/yky/blog/api/service/impl/CollectionApiServiceImpl.java
```

接口建议：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/collection/list` | 合集列表 |
| GET | `/api/collection/{id}/articles` | 合集下文章分页 |

合集列表建议补充每个合集下的已发布文章数量。MVP 可用简单查询实现，不必一开始追求复杂 SQL。

### 6.6 后端注意事项

1. 前台接口不要复用 `/admin/article` 路径。
2. 所有前台文章查询必须加 `status = 1`。
3. 文章详情查不到或文章未发布时，返回“文章不存在”。
4. 列表接口可以复用后台组装合集、标签的思路。
5. 阅读量本阶段可以先不自增，或先简单 MySQL `view_count + 1`，Redis 异步统计后续单独做。
6. 归档接口可以先查所有已发布文章，再按 `yyyy-MM` 在 Java 中分组。

## 七、前端开发计划

### 7.1 清理 Vite 初始页

当前 `blog-web/src/App.vue` 仍然是：

```vue
<HelloWorld />
```

下一步应删除对 `HelloWorld` 的依赖，建立真实前台应用结构。

建议目录：

```text
blog-web/src/router/index.ts
blog-web/src/utils/request.ts
blog-web/src/api/article.ts
blog-web/src/api/tag.ts
blog-web/src/api/collection.ts
blog-web/src/layout/index.vue
blog-web/src/views/home/index.vue
blog-web/src/views/article/detail.vue
blog-web/src/views/tag/index.vue
blog-web/src/views/tag/articles.vue
blog-web/src/views/collection/index.vue
blog-web/src/views/collection/detail.vue
blog-web/src/views/archive/index.vue
blog-web/src/views/about/index.vue
```

### 7.2 安装前台必要依赖

建议前台先使用轻量依赖：

```bash
npm install axios vue-router md-editor-v3
```

如果后续需要状态管理再加 Pinia。前台 MVP 阶段不需要复杂全局状态。

### 7.3 路由设计

建议路由：

| 路径 | 页面 |
| --- | --- |
| `/` | 首页文章列表 |
| `/article/:id` | 文章详情 |
| `/tags` | 标签列表 |
| `/tag/:id` | 标签文章列表 |
| `/collections` | 合集列表 |
| `/collection/:id` | 合集详情 |
| `/archives` | 归档页 |
| `/about` | 关于我 |

### 7.4 页面开发顺序

前端建议按这个顺序做：

1. 搭 `layout`：顶部导航、主内容区、页脚。
2. 搭 `request.ts`：统一 baseURL 和错误提示。
3. 搭 `api/article.ts`：先接文章分页和详情。
4. 做首页文章列表。
5. 做文章详情页。
6. 做标签列表和标签文章列表。
7. 做合集列表和合集详情。
8. 做归档页。
9. 做移动端响应式适配。

### 7.5 首页文章列表

首页先保证阅读入口清晰，不要做成营销落地页。

文章卡片建议展示：

- 封面
- 标题
- 摘要
- 标签
- 合集
- 发布时间
- 阅读量

无封面时可以使用简洁占位，不要让列表布局塌陷。

### 7.6 文章详情页

详情页重点是阅读体验。

建议展示：

- 标题
- 发布时间
- 标签
- 合集
- 封面
- 正文
- 原创 / 转载信息

正文渲染策略：

```text
MVP：前端用 md-editor-v3 的 previewOnly 渲染 contentMd
后续：后台保存 HTML，前台优先渲染 content
```

### 7.7 响应式要求

MVP 至少覆盖：

- 桌面端 1440px
- 笔记本 1280px
- 平板 768px
- 手机 375px

移动端不要保留过宽侧栏。文章列表、合集列表、标签列表都应切成单列。

## 十一、阶段完成后的下一步

前台展示 MVP 完成后，建议进入：

**阶段 6：内容体验增强**

优先级建议：

1. 图片上传：支撑 Markdown 编辑器插图。
2. 网站配置：站点名、作者信息、关于我、社交链接。
3. 轮播管理：首页推荐内容。
4. 评论系统：读者互动。

6. 搜索：先 MySQL 关键词搜索，再考虑全文索引。
7. AI 摘要 / 标签推荐：等内容生产和展示稳定后再接入。
