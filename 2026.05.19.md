### 1.3 业务异常类

**文件路径**：`common/exception/BizException.java`

**思路**：业务层抛出此异常，由全局处理器统一捕获，避免在 Controller 里写 try-catch。

```java
package com.yky.blog.common.exception;

import com.yky.blog.common.result.ResultCode;
import lombok.Getter;

@Getter
public class BizException extends RuntimeException {

    private final int code;

    public BizException(String message) {
        super(message);
        this.code = ResultCode.FAILED.getCode();
    }

    public BizException(ResultCode resultCode) {
        super(resultCode.getMessage());
        this.code = resultCode.getCode();
    }

    public BizException(ResultCode resultCode, String message) {
        super(message);
        this.code = resultCode.getCode();
    }
}
```

### 1.4 全局异常处理器

**文件路径**：`common/exception/GlobalExceptionHandler.java`

**思路**：用 `@RestControllerAdvice` 拦截三类异常：
- `BizException`：业务主动抛出的
- `MethodArgumentNotValidException`：参数校验（`@Valid`）失败
- `Exception`：兜底，防止未知异常把堆栈信息返回给前端

```java
package com.yky.blog.common.exception;

import com.yky.blog.common.result.Result;
import com.yky.blog.common.result.ResultCode;
import lombok.extern.slf4j.Slf4j;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import java.util.stream.Collectors;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BizException.class)
    public Result<Void> handleBizException(BizException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.failed(ResultCode.FAILED, e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors()
                .stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining(", "));
        return Result.failed(ResultCode.VALIDATE_FAILED, message);
    }

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常: ", e);
        return Result.failed("系统繁忙，请稍后再试");
    }
}
```

### 1.6 Sa-Token 配置

**文件路径**：`common/config/SaTokenConfig.java`

**思路**：Sa-Token 通过拦截器检查 Token，需要注册到 Spring MVC。
放行路径（登录接口、文档接口）必须配置，否则自己登录都会被拦截。

```java
package com.yky.blog.common.config;

import cn.dev33.satoken.interceptor.SaInterceptor;
import cn.dev33.satoken.stp.StpUtil;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class SaTokenConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new SaInterceptor(handle -> StpUtil.checkLogin()))
                .addPathPatterns("/**")
                .excludePathPatterns(
                        "/admin/auth/login",
                        "/api/**",
                        "/doc.html",
                        "/webjars/**",
                        "/swagger-resources/**",
                        "/v3/api-docs/**"
                );
    }
}
```

## 第二步：实体类 + Mapper

### 思路

MyBatis Plus 实体类有几个固定写法：
- `@TableName("表名")`：指定对应的数据库表
- `@TableId(type = IdType.AUTO)`：主键自增
- `@TableField(fill = FieldFill.INSERT)`：自动填充创建时间

Mapper 只需继承 `BaseMapper<T>`，CRUD 方法全部自动提供，不用写任何 SQL。

### 2.1 公共基类

**文件路径**：`common/entity/BaseEntity.java`

大多数表都有 `create_time` / `update_time`，抽一个基类减少重复代码。

```java
package com.yky.blog.common.entity;

import com.baomidou.mybatisplus.annotation.TableField;
import lombok.Data;

import java.time.LocalDateTime;

@Data

    private LocalDateTime createTime;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;

```

要让自动填充生效，还需要一个填充处理器：

**文件路径**：`common/config/MyBatisPlusHandler.java`

```java
package com.yky.blog.common.config;

import com.baomidou.mybatisplus.core.handlers.MetaObjectHandler;
import org.apache.ibatis.reflection.MetaObject;
import org.springframework.stereotype.Component;

import java.time.LocalDateTime;

@Component
public class MyBatisPlusHandler implements MetaObjectHandler {

    @Override
    public void insertFill(MetaObject metaObject) {
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
}
```

## 第三步：管理员登录

### 思路

登录接口是整个后台系统的入口，完成它之后你就可以：
- 用 Knife4j 文档测试所有需要鉴权的接口
- 开始联调 Vue3 admin 前端

登录流程：
```
前端 POST /admin/auth/login
  → Controller 接收 DTO，参数校验
  → Service 查用户名，判断状态
  → BCrypt 验证密码
  → Sa-Token 签发 Token
  → 返回 Token + 用户基本信息
```

### 3.4 Service 实现

**文件路径**：`auth/service/impl/AdminServiceImpl.java`

**思路逐步解释**：
1. 用 MyBatis Plus 的 `LambdaQueryWrapper` 按用户名查，查不到直接抛业务异常
2. 判断账号状态，被禁用的不让登录
3. `BCrypt.checkpw()` 验证密码（数据库存的是哈希，不能明文对比）
4. `StpUtil.login(id)` 完成 Sa-Token 登录，内部自动生成并存储 Token
5. `StpUtil.getTokenValue()` 取出刚签发的 Token 返回给前端

```java
package com.yky.blog.auth.service.impl;

import cn.dev33.satoken.stp.StpUtil;
import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.yky.blog.auth.dto.LoginDTO;
import com.yky.blog.auth.service.AdminService;
import com.yky.blog.auth.vo.LoginVO;
import com.yky.blog.common.entity.Admin;
import com.yky.blog.common.exception.BizException;
import com.yky.blog.common.mapper.AdminMapper;
import lombok.RequiredArgsConstructor;
import org.springframework.security.crypto.bcrypt.BCrypt;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class AdminServiceImpl implements AdminService {

    private final AdminMapper adminMapper;

    @Override
    public LoginVO login(LoginDTO dto) {
        // 1. 查用户名
        Admin admin = adminMapper.selectOne(
                new LambdaQueryWrapper<Admin>()
                        .eq(Admin::getUsername, dto.getUsername())
        );
        if (admin == null) {
            throw new BizException("用户名或密码错误");
        }

        // 2. 判断账号状态
        if (admin.getStatus() == 0) {
            throw new BizException("账号已被禁用，请联系超级管理员");
        }

        // 3. 验证密码（BCrypt）
        if (!BCrypt.checkpw(dto.getPassword(), admin.getPassword())) {
            throw new BizException("用户名或密码错误");
        }

        // 4. Sa-Token 登录，签发 Token
        StpUtil.login(admin.getId());

        // 5. 返回 Token 和基本信息
        return LoginVO.builder()
                .token(StpUtil.getTokenValue())
                .id(admin.getId())
                .nickname(admin.getNickname())
                .build();
    }

    @Override
    public void logout() {
        StpUtil.logout();
    }
}
```

> **关于 BCrypt 依赖**：需在 `pom.xml` 中添加（不引入整个 Spring Security）：
>
> ```xml
> <dependency>
>     <groupId>org.springframework.security</groupId>
>     <artifactId>spring-security-crypto</artifactId>
> </dependency>
> ```

package com.yky.blog.auth.controller;
