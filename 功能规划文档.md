# kyonelife 博客系统功能规划文档

> 技术栈：Spring Boot 3.x + Vue 3 + MySQL 8.0 + Redis
> 原则：先跑通核心流程，再按优先级迭代扩展

---

## 一、用户与权限体系

### 1.1 前台访客用户
- 手机号 + 短信验证码登录（无需注册，首次登录自动建档）
- 自动生成昵称（如 `用户_3k9x`），无需用户填写
- 登录后可发表评论、回复评论
- 记录最后登录 IP、IP 归属地、浏览器、操作系统

### 1.2 后台管理员
- 账号 + 密码登录（BCrypt 加密）
- 记录最后登录 IP、IP 归属地、浏览器、操作系统

### 1.3 RBAC 权限系统
- **角色表**：角色编码（SUPER / REGULAR）+ 角色名称
- **菜单表**：三级结构（目录 → 菜单 → 按钮），按钮级权限标识如 `article:add`
- **角色菜单关联**：灵活配置每个角色能访问哪些菜单和按钮
- 内置两个角色：
  - `SUPER` 超级管理员：拥有全部权限
  - `REGULAR` 普通管理员：只读，只有 `*:list` 权限
- 后续可在后台自由新增角色、自定义分配权限，无需改代码

## 二、文章系统

### 2.1 基础功能
- Markdown 编辑器写作，同时存储 HTML 内容和 Markdown 原文
- 文章状态：草稿 / 已发布 / 已下架
- 文章封面图（单图）
- 文章摘要（手动填写或 AI 生成）
- 原创 / 转载标记，转载需填写原文链接
- SEO 关键词字段
- 阅读量统计（Redis 缓存 + 定时刷库）

### 2.2 合集功能
- 创建合集（名称、封面、简介、排序）
- 文章可加入合集，也可不加入
- 未加入合集的文章直接出现在文章列表
- 合集详情页展示该合集下所有文章

### 2.3 标签功能
- 写文章时输入新标签名自动创建
- 也可从已有标签中直接选择
- 标签支持颜色设置
- 一篇文章可以有多个标签

### 2.4 置顶与轮播
- 文章可设置置顶，置顶文章排在列表最前
- 文章可加入首页轮播，最多 5 篇
- 轮播文章数量为 0 时，自动切换为图片轮播（banner 表）
- 轮播文章支持排序

### 2.5 文章多图
- 文章内容中支持插入多张图片
- 图片独立存储在 `article_image` 表，支持排序

## 三、评论系统

- 必须登录（手机号验证码）才能评论
- 支持回复评论（二级嵌套）
- 记录评论者 IP、IP 归属地、浏览器、操作系统、评论时间
- 后台可隐藏或删除评论
- 网站配置中可全局开关评论功能

---

## 四、后台管理功能

### 内容管理
- 文章管理：列表、新增、编辑、删除、状态修改、置顶设置、轮播设置
- 合集管理：增删改查
- 标签管理：增删改查
- 评论管理：列表查看、隐藏、删除
- 轮播图管理：图片轮播的增删改查、排序

### 系统管理
- 管理员管理：新增、编辑、禁用管理员账号
- 角色管理：新增角色、分配菜单权限
- 菜单管理：查看权限菜单树
- 用户管理：查看前台用户列表、封禁用户
- 网站配置：网站名称、作者信息、社交链接、ICP 备案号、公告内容、是否开启评论

---

## 五、前台展示功能

- 首页：文章列表（置顶优先）、轮播图/轮播文章
- 文章详情页：内容渲染、代码高亮、评论区
- 合集页：合集列表、合集内文章列表
- 标签页：所有标签、点击标签筛选文章
- 归档页：按年月分组展示文章
- 关于我页：Markdown 内容渲染
- 响应式设计，适配手机端

## 六、AI 功能规划

### MVP 阶段（优先实现）

| 功能 | 说明 | 对应字段 |
|------|------|---------|
| 自动生成摘要 | 文章写完后一键调用大模型生成摘要 | `article.summary` |
| AI 文章描述 | 生成更简短的一句话描述 | `article.ai_describe` |
| 智能标签推荐 | 根据文章内容推荐相关标签，人工确认后保存 | `article_tag` |
| SEO 关键词提取 | 自动提取并填充 keywords 字段 | `article.keywords` |

### 进阶阶段

| 功能 | 说明 | 技术点 |
|------|------|--------|
| 文章润色/纠错 | 编辑器内划选文字，AI 优化表达 | 调用大模型 API |
| 评论内容审核 | AI 自动识别垃圾评论，标记为待审核 | 文本分类 |
| 评论情感分析 | 统计读者对文章的整体情感倾向 | 情感分析模型 |
| 相关文章推荐 | 基于向量相似度推荐语义相近的文章 | Embedding + 向量检索 |

### 高阶阶段（简历亮点）

| 功能 | 说明 | 技术架构 |
|------|------|---------|
| **AI 文章问答（RAG）** | 访客针对当前文章提问，AI 基于文章内容回答 | 文章向量化 → 向量数据库 → 召回 → 大模型生成答案 |
| **智能语义搜索** | 输入"讲缓存的文章"也能搜到，不只是关键词匹配 | Embedding + 向量相似度检索 |
| **写作方向建议** | 根据已有文章标签分布，建议下一篇写什么方向 | 数据统计 + 大模型建议 |

## 七、技术深度功能（简历加分项）

### 7.1 缓存架构
- Redis 多级缓存：热门文章、首页数据、用户 Session
- 缓存击穿（互斥锁）、雪崩（随机过期时间）、穿透（布隆过滤器）处理
- 阅读量先写 Redis，定时任务批量刷入 MySQL

### 7.2 全文搜索
- MVP 阶段：MySQL 全文索引（FULLTEXT）
- 进阶：接入 Elasticsearch，支持高亮显示、分词搜索

### 7.3 接口安全
- 评论、验证码接口基于 Redis 滑动窗口限流
- 防止短时间内大量请求刷接口

### 7.4 CI/CD 自动化部署
- GitHub Actions：推送代码自动触发构建、测试、部署
- Docker Compose 管理服务（MySQL、Redis、后端、前端）

### 7.5 埋点与数据统计
- 记录文章访问日志（PV/UV、来源、停留时长）
- 记录用户行为事件（点击、搜索）
- 后台数据看板（ECharts 可视化）
- 不影响现有表结构，独立新增 `visit_log` 和 `event_log` 两张表

### 7.6 文件存储
- MVP 阶段：本地存储
- 进阶：接入阿里云 OSS / 腾讯云 COS，配置可切换

## 八、开发优先级

| 阶段 | 功能 | 预计时间 |
|------|------|---------|
| **第一阶段（MVP）** | 管理员登录、文章增删改查、标签、合集、评论、前台展示 | 4～6 周 |
| **第二阶段** | 手机号登录、轮播、网站配置、图片上传、限流 | 2 周 |
| **第三阶段** | AI 摘要/标签推荐、Redis 缓存优化、埋点统计 | 2 周 |
| **第四阶段** | CI/CD、Elasticsearch 搜索、AI 问答（RAG） | 按需迭代 |
