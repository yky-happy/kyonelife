# 个人博客系统开发计划

> 技术栈：Java 后端 + Vue 前端，参考 shiyi-blog 从头开发
>
> 原则：先跑通核心流程，再逐步补充功能，不要一开始就照搬 shiyi-blog 的全部功能

---

## 第一步：想清楚自己要什么（需求梳理）

开发之前最重要的一件事：**列出你自己真正需要的功能**，不需要的不做。

shiyi-blog 功能很全，但作为个人博客很多是用不上的，比如实时聊天室、代码生成、说说、资源下载等。先问自己几个问题：

| 问题 | 参考选择 |
|------|---------|
| 博客只有你写，还是允许别人注册投稿？ | 个人博客：只有你自己写 |
| 需要评论功能吗？允许游客评论还是登录才能评论？ | 建议保留，游客可评论 |
| 需要文章分类和标签吗？ | 建议保留 |
| 需要相册功能吗？ | 按需 |
| 前台需要用户登录吗？ | 纯展示博客可以不需要 |
| 需要移动端 App 吗？ | 做好响应式网页即可，不必单独做 App |

**建议的最小功能集（MVP）：**

- 文章发布、编辑、删除（Markdown 编辑）
- 文章 标签 + 合集功能
- 文章归档
- 评论（游客可评论，记录 IP）
- 后台管理（只有你能登录）
- 首页、文章详情页、关于我页

## 第二步：技术选型

### 后端

| 技术 | 选择 | 说明 |
|------|------|------|
| 框架 | Spring Boot 3.x | 比 shiyi-blog 的 2.7 更新，支持 JDK 17+ |
| ORM | MyBatis Plus | 和 shiyi-blog 一致，减少 SQL 编写量 |
| 认证 | Sa-Token 或 Spring Security | Sa-Token 更简单，建议沿用 |
| 数据库 | MySQL 8.0 | 稳定，全文索引支持更好 |
| 缓存 | Redis | 验证码、限流、统计计数 |
| 构建 | Maven | 和 shiyi-blog 一致 |
| API 文档 | Knife4j（Swagger 增强版） | 方便前后端联调 |

### 前端

| 技术 | 选择 | 说明 |
|------|------|------|
| 框架 | Vue 3 + TypeScript | shiyi-blog 后台已用 Vue 3，统一技术栈 |
| UI 组件库 | Element Plus（后台）/ 自定义或 Naive UI（前台） | 后台用 Element Plus 省事 |
| 构建工具 | Vite | 快 |
| 状态管理 | Pinia | Vue 3 官方推荐 |
| 编辑器 | Vditor 或 ByteMD | Markdown 编辑器，比 shiyi-blog 的 Mavon Editor 更现代 |
| HTTP 请求 | Axios | 标准选择 |

### 项目结构规划

```
my-blog/
├── blog-backend/          # Spring Boot 后端
│   ├── src/main/java/
│   │   ├── controller/    # 接口层
│   │   ├── service/       # 业务层
│   │   ├── mapper/        # 数据库层
│   │   ├── entity/        # 实体类
│   │   ├── dto/           # 请求参数对象
│   │   ├── vo/            # 响应数据对象

│   └── src/main/resources/
│       ├── mapper/        # MyBatis XML
│       └── application.yml
├── blog-admin/            # 后台管理前端（Vue 3）
└── blog-web/              # 前台展示前端（Vue 3）
```

## 第三步：搭建开发环境

### 3.1 后端环境

```bash
# 安装 JDK 17
# 安装 Maven 3.8+
# 安装 MySQL 8.0
# 安装 Redis 7.x
```

创建数据库：
```sql
CREATE DATABASE my_blog DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 创建 Spring Boot 项目

去 [start.spring.io](https://start.spring.io) 生成，选择以下依赖：
- Spring Web
- MySQL Driver
- Spring Data Redis
- Lombok
- Validation

然后在 `pom.xml` 手动添加：
```xml
<!-- MyBatis Plus -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.5</version>
</dependency>
<!-- Sa-Token -->
<dependency>
    <groupId>cn.dev33</groupId>
    <artifactId>sa-token-spring-boot3-starter</artifactId>
    <version>1.37.0</version>
</dependency>
<!-- Knife4j -->
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.4.0</version>
</dependency>
```

### 3.3 配置 application.yml

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/my_blog?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: yourpassword
  data:
    redis:
      host: localhost
      port: 6379

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0

sa-token:
  token-name: Authorization
  timeout: 2592000   # 30天
  is-concurrent: false
```

### 3.4 创建前端项目

```bash
# 后台管理
npm create vite@latest blog-admin -- --template vue-ts
cd blog-admin && npm install
npm install element-plus axios pinia vue-router

# 前台展示
npm create vite@latest blog-web -- --template vue-ts
cd blog-web && npm install
npm install axios pinia vue-router
```

## 第五步：后端开发

按以下顺序开发，每完成一个模块都用 Knife4j 测试接口。

### 5.1 基础框架搭建（第1周）

- [ ] 统一响应体封装 `Result<T>`
- [ ] 全局异常处理 `GlobalExceptionHandler`
- [ ] MyBatis Plus 配置（分页插件、自动填充 create_time/update_time）
- [ ] Redis 工具类 `RedisUtil`
- [ ] 跨域配置 `CorsConfig`

参考 shiyi-blog：
- `mojian-commom/utils/RedisUtil.java`
- `mojian-commom/config/RedisConfig.java`

---

### 5.2 认证模块（第1周）

**目标：** 后台登录、登出、获取当前用户信息

接口：
```
POST /auth/login      # 账号密码登录
POST /auth/logout     # 登出
GET  /auth/info       # 获取当前登录信息
```

实现要点：
- 密码用 `BCryptPasswordEncoder` 加密存储
- 登录成功后用 Sa-Token `StpUtil.login(userId)` 生成 token
- 需要登录的接口加 `@SaCheckLogin` 注解
- 初始化一条管理员账号数据

参考 shiyi-blog：
- `mojian-auth/service/impl/AuthServiceImpl.java` 的 login 方法

---

### 5.3 文章模块（第2周）

**后台接口：**
```
GET    /sys/article/list           # 文章列表（分页）
GET    /sys/article/detail/{id}    # 文章详情
POST   /sys/article/add            # 新增文章
PUT    /sys/article/update         # 修改文章
PUT    /sys/article/updateStatus   # 修改状态（发布/草稿/置顶）
DELETE /sys/article/delete/{ids}   # 删除（批量）
```

**前台接口：**
```
GET /api/article/list              # 文章列表（分页，支持分类/标签/关键词）
GET /api/article/detail/{id}       # 文章详情（阅读量+1）
GET /api/article/archive           # 归档
```

实现要点：
- 新增/修改文章时同步处理 `article_tag` 关联表（先删后插）
- 阅读量先写 Redis Hash，由定时任务每10分钟刷入 MySQL（参考 shiyi-blog 的 `ArticleServiceImpl`）
- 关键词搜索用 MySQL 全文索引 `MATCH(title) AGAINST(?)`

参考 shiyi-blog：
- `mojian-api/service/impl/ArticleServiceImpl.java`
- `mojian-admin/service/impl/SysArticleServiceImpl.java`

---

### 5.4 分类和标签模块（第2周）

**接口（后台）：**
```
GET/POST/PUT/DELETE /sys/category/*
GET/POST/PUT/DELETE /sys/tag/*
```

逻辑简单，CRUD 即可。注意删除分类时检查是否有文章在用。

---

### 5.5 评论模块（第3周）

**接口：**
```
GET  /api/comment/list   # 按文章ID获取评论（分页，支持按时间/热度排序）
POST /api/comment/add    # 发表评论（游客填昵称+邮箱）
GET  /sys/comment/list   # 后台评论管理
DELETE /sys/comment/delete/{ids}
```

实现要点：
- 游客评论不需要登录，只需填昵称
- 自动记录 IP 和 IP 归属地（使用 `ip2region` 库）
- 评论支持回复（`parent_id` + `reply_nickname`）
- 前台展示时先查顶级评论，再查每条评论的回复

---

### 5.6 文件上传模块（第3周）

**接口：**
```
POST /sys/file/upload    # 上传文件，返回访问 URL
```

实现要点：
- MVP 阶段直接用**本地存储**（存到服务器目录，配置静态资源访问）
- 后期可扩展到阿里云 OSS 等（参考 shiyi-blog 的 `mojian-file` 模块）
- 验证文件类型和大小

### 5.7 网站配置模块（第4周）

**接口：**

```
GET /api/home/config     # 前台获取配置（网站名、作者信息等）
GET /sys/web/config      # 后台获取配置
```

## 第六步：前端开发

### 6.1 后台管理（blog-admin）

页面开发顺序（优先级从高到低）：

- [ ] **登录页** → 调用 `/auth/login`，token 存 localStorage，Axios 拦截器自动带 token
- [ ] **布局框架** → 侧边栏导航 + 顶部 + 内容区（用 Element Plus 的 Layout）
- [ ] **文章列表页** → 表格展示、分页、搜索、操作按钮
- [ ] **文章编辑页** → 核心页面，集成 Markdown 编辑器（推荐 [Vditor](https://b3log.org/vditor/)）
- [ ] **分类管理页** → 简单表格 + 弹窗表单
- [ ] **标签管理页** → 同上
- [ ] **评论管理页** → 表格 + 删除
- [ ] **网站配置页** → 表单

Axios 拦截器统一处理：
```typescript
// request.ts
instance.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers['Authorization'] = token
  return config
})

instance.interceptors.response.use(
  res => res.data,
  err => {
    if (err.response?.status === 401) router.push('/login')
    return Promise.reject(err)
  }
)
```

### 6.2 前台展示（blog-web）

页面开发顺序：

- [ ] **首页** → 文章列表卡片（封面、标题、摘要、分类、时间）、分页
- [ ] **文章详情页** → Markdown 渲染（推荐 `md-it` 或 `marked` + `highlight.js` 代码高亮）
- [ ] **评论区组件** → 可复用的评论列表 + 发表评论表单
- [ ] **归档页** → 按年月分组展示文章
- [ ] **分类/标签筛选** → 点击分类/标签过滤文章列表
- [ ] **关于我页** → 渲染 Markdown 内容
- [ ] **404 页面**

前台不需要登录功能，响应式设计适配手机端（用 CSS 媒体查询或 Tailwind CSS）。

## 第八步：部署上线

### 8.1 服务器准备

推荐使用云服务器（阿里云/腾讯云），最低配置：2核4G，带宽3Mbps。

安装：
```bash
# JDK 17
apt install openjdk-17-jdk

# MySQL 8.0
apt install mysql-server

# Redis
apt install redis-server

# Nginx
apt install nginx
```

### 8.2 后端打包部署

```bash
# 本地打包
mvn clean package -DskipTests

# 上传 jar 包到服务器
scp target/my-blog.jar user@server:/app/

# 服务器上运行
nohup java -jar /app/my-blog.jar --spring.profiles.active=prod > /app/logs/app.log 2>&1 &
```

### 8.3 前端打包部署

```bash
# 本地打包
npm run build

# 上传 dist 目录到服务器
scp -r dist/ user@server:/var/www/blog-web/
scp -r dist/ user@server:/var/www/blog-admin/
```

### 8.4 Nginx 配置

```nginx
# 前台展示
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        root /var/www/blog-web;
        index index.html;
        try_files $uri $uri/ /index.html;  # Vue Router history 模式
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
    }
}

# 后台管理
server {
    listen 80;
    server_name admin.yourdomain.com;

    location / {
        root /var/www/blog-admin;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /sys/ {
        proxy_pass http://127.0.0.1:8080/sys/;
    }

    location /auth/ {
        proxy_pass http://127.0.0.1:8080/auth/;
    }
}
```

### 8.5 配置 HTTPS（推荐）

使用 Let's Encrypt 免费证书：
```bash
apt install certbot python3-certbot-nginx
certbot --nginx -d yourdomain.com -d admin.yourdomain.com
```

## 常见坑提醒

1. **前后端跨域**：开发阶段用 Vite 的 proxy 配置解决，生产环境走 Nginx 反向代理，不要在 Nginx 和 Spring Boot 里同时配置 CORS
2. **Markdown 渲染 XSS**：前台渲染用户输入的 Markdown 时必须做 HTML 过滤（用 `DOMPurify`）
3. **文件上传路径**：本地存储时注意 Windows/Linux 路径分隔符差异，路径用变量配置不要硬编码
4. **Redis 序列化**：RedisTemplate 默认序列化方式读起来乱码，要配置 Jackson 序列化（参考 shiyi-blog 的 `RedisConfig.java`）
5. **密码安全**：绝对不能存明文密码，一定要用 BCrypt
6. **SQL 注入**：MyBatis Plus 的 `${}`  是拼接 SQL 有注入风险，用 `#{}` 替代
