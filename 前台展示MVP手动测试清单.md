# 前台展示 MVP 手动测试清单

## 一、测试目标

本次测试的目标是确认项目已经跑通最小博客闭环：

```text
后台登录 -> 写文章 -> 发布文章 -> 前台首页看到文章 -> 点击进入文章详情
```

同时确认标签、合集、归档和草稿/下架过滤这些前台展示 MVP 的核心功能可用。

## 二、测试前准备

### 1. 启动后端

进入后端目录：

```bash
cd blog
```

启动 Spring Boot 项目。

确认后端正常启动后，访问接口文档或接口地址时不应出现服务不可用。

### 2. 启动后台管理端

进入后台前端目录：

```bash
cd blog-admin
npm run dev
```

打开后台管理端页面，确认登录页可以访问。

### 3. 启动前台展示端

进入前台前端目录：

```bash
cd blog-web
npm run dev
```

打开前台页面，确认页面可以正常显示，不再是 Vite 初始页面。

### 4. 准备测试数据

至少准备以下数据：

- 1 个标签，例如：`Java`
- 1 个标签，例如：`Vue`
- 1 个合集，例如：`项目复盘`
- 1 篇已发布文章，带标题、摘要、正文、标签、合集
- 1 篇草稿文章
- 1 篇下架文章

## 三、后台文章链路测试

### 1. 登录后台

测试内容：

- 输入管理员账号密码
- 点击登录
- 登录成功后进入后台首页

通过标准：

- 登录成功后能进入后台管理页面
- 页面刷新后仍能保持登录状态
- 退出登录后不能继续访问后台受保护页面

### 2. 新建文章草稿

测试内容：

- 进入文章管理
- 点击新增文章
- 填写标题、摘要、正文
- 选择标签
- 选择合集
- 状态选择草稿
- 保存

通过标准：

- 保存成功
- 文章列表能看到该草稿
- 草稿状态显示正确

### 3. 编辑文章

测试内容：

- 打开刚才创建的文章
- 修改标题、摘要、正文
- 修改标签或合集
- 保存

通过标准：

- 保存成功
- 回到列表后修改内容正确显示
- 再次进入编辑页，内容能正确回显

### 4. 发布文章

测试内容：

- 将一篇草稿文章改为已发布
- 保存或点击发布操作

通过标准：

- 文章状态变为已发布
- 前台首页能看到这篇文章

### 5. 下架文章

测试内容：

- 将一篇已发布文章改为下架

通过标准：

- 后台列表状态变为下架
- 前台首页不再展示这篇文章
- 直接访问该文章详情地址时，应提示文章不存在或加载失败

## 四、前台首页测试

### 1. 首页文章列表

测试内容：

- 打开前台首页 `/`
- 查看文章列表

通过标准：

- 只展示已发布文章
- 不展示草稿文章
- 不展示下架文章
- 文章标题、摘要、封面、标签、合集、发布时间能正常显示
- 没有封面时，占位样式正常，不影响布局

### 2. 置顶排序

测试内容：

- 后台设置一篇文章为置顶
- 返回前台首页刷新

通过标准：

- 置顶文章排在普通文章前面
- 置顶标识正常显示

### 3. 加载更多

测试内容：

- 准备超过一页数量的已发布文章
- 点击首页的加载更多按钮

通过标准：

- 能继续加载下一页文章
- 已加载文章不会重复
- 没有更多文章时，不应继续显示可加载状态

## 五、文章详情页测试

### 1. 从首页进入详情

测试内容：

- 在首页点击一篇文章

通过标准：

- 能跳转到 `/article/:id`
- 标题、摘要、发布时间、阅读量、原创/转载状态正常显示
- 标签正常显示
- 封面正常显示
- 正文正常渲染

### 2. Markdown 正文渲染

测试内容：

在文章正文中测试以下内容：

- 一级标题、二级标题
- 加粗、斜体
- 列表
- 代码块
- 链接
- 图片地址

通过标准：

- Markdown 内容能正常转换为页面内容
- 代码块、列表、链接不破坏页面布局
- 图片过宽时不撑破页面

### 3. 转载文章来源

测试内容：

- 创建一篇转载文章
- 填写原文链接
- 发布后进入前台详情页

通过标准：

- 页面显示转载来源
- 点击来源链接能打开新页面

### 4. 不可见文章详情

测试内容：

- 直接访问草稿文章详情地址
- 直接访问下架文章详情地址
- 直接访问不存在的文章 ID

通过标准：

- 页面不显示文章正文
- 应显示错误提示，例如文章不存在或文章加载失败

## 六、标签功能测试

### 1. 标签列表页

测试内容：

- 打开 `/tags`

通过标准：

- 标签列表正常显示
- 标签名称、颜色、文章数量正常显示
- 点击标签能进入对应标签文章页

### 2. 标签文章页

测试内容：

- 打开 `/tag/:id`

通过标准：

- 页面标题显示当前标签名称
- 只展示该标签下的已发布文章
- 不展示草稿和下架文章
- 点击文章能进入详情页

### 3. 空标签测试

测试内容：

- 准备一个没有已发布文章的标签
- 打开该标签文章页

通过标准：

- 页面显示空状态提示
- 页面不报错

通过标准：

- 只展示该合集下的已发布文章
- 不展示草稿和下架文章
- 点击文章能进入详情页

### 3. 空合集测试

测试内容：

- 准备一个没有已发布文章的合集
- 打开该合集详情页

通过标准：

- 页面显示空状态提示

## 八、归档页测试

### 1. 归档列表

测试内容：

- 打开 `/archives`

通过标准：

- 已发布文章按年月分组
- 同一月份下文章按发布时间倒序显示
- 点击文章能进入详情页

### 2. 归档过滤

测试内容：

- 准备草稿文章和下架文章
- 打开归档页

通过标准：

- 草稿文章不出现在归档页
- 下架文章不出现在归档页

## 九、前台导航测试

测试内容：

- 点击首页导航
- 点击标签导航
- 点击合集导航
- 点击归档导航
- 点击关于页面
- 从详情页返回首页

通过标准：

- 所有导航都能正常跳转
- 页面刷新后当前路由仍能正常访问
- 不出现空白页或控制台明显报错

## 十、响应式测试

至少测试以下宽度：

- 桌面端：1440px
- 笔记本：1280px
- 平板：768px
- 手机：375px

重点检查：

- 页面没有横向滚动条
- 文章卡片在手机端变成单列
- 顶部导航在手机端不挤压重叠
- 文章详情正文不超出屏幕
- 图片不会撑破页面
- 按钮文字不溢出

## 十一、接口可见性测试

可以直接在浏览器或接口工具中访问：

```text
GET /api/article/page
GET /api/article/page?tagId=1
GET /api/article/page?collectionId=1
GET /api/article/{id}
GET /api/article/archive
GET /api/tag/list
GET /api/collection/list
GET /api/collection/{id}/articles
```

重点确认：

- `/api/article/page` 只返回已发布文章
- `/api/article/{id}` 不能访问草稿文章
- `/api/article/{id}` 不能访问下架文章
- 标签筛选结果正确
- 合集筛选结果正确
- 归档结果只包含已发布文章

## 十三、通过标准

本阶段可以认为通过的条件：

- 后台能创建、编辑、发布、下架文章
- 前台首页只展示已发布文章
- 文章详情页能正常阅读正文
- 标签、合集、归档都能正常进入文章详情
- 草稿和下架文章不会出现在任何前台列表中
- 手机端没有明显布局错乱

如果以上条件满足，前台展示 MVP 就可以进入下一阶段：评论系统。
